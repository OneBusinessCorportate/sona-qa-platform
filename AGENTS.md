# AGENTS.md — правила работы в этом репозитории (для ИИ-агентов и ИИ-инженеров)

Канон процесса — `OneBusinessCorportate/.github/dev-process.md`. Здесь минимум, обязательный к соблюдению.

## Ветка `main` защищена
Прямой push в `main` запрещён. Порядок:

1. Ветка от `main` (`issue-<N>` или осмысленное имя).
2. Изменил код — **в том же PR** добавь запись в `CHANGELOG.md`: что / зачем / результат.
3. Открой PR в `main`. Робот `doc-sync` (проверка `changelog`) обязан быть зелёным — на красном кнопка мерджа физически заблокирована для всех, включая владельца.
4. Заведомо тривиальная правка — метка PR `skip-changelog`.

## Ревью
Правило «автор ≠ ревьюер»: одобрять и мерджить свой PR той же сессией/агентом **нельзя**. Независимый вердикт выносит внешняя модель (codex/GPT) — детали в каноне.
