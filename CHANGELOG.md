# Журнал изменений

Запись формата «что / зачем / результат» обязательна для каждого PR, меняющего код: без неё робот `doc-sync` краснит и мердж заблокирован. Форточка для заведомо тривиальной правки — метка PR `skip-changelog`.

## 2026-07-16
- «Ответы бухгалтеров»: добавлен переключатель «Показать закрытые» (`GET /tickets/feed?closed=1`) и исправлен текст пустого состояния. Зачем: лента скрывала обработанные тикеты (`explained_accepted`/`fixed`/`auto_resolved`), поэтому реально отправленные ответы бухгалтеров (формы + комментарии) были не видны Соне, а сообщение ложно гласило «ничего не отправляли». Результат: по умолчанию видны только открытые диалоги, но Сона может включить показ закрытых и увидеть все ответы; пустой текст теперь различает «нет открытых» и «ничего не отправляли».

## 2026-07-14
- Заведён журнал изменений, коммит-gate (`doc-sync`) и защита ветки `main`. Зачем: изменение кода не должно попадать в продукт без следа «что поменяли». Результат: работа только через PR, мердж на красном физически заблокирован для всех.
